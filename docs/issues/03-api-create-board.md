## Wire Contract

**Mandatory endpoints:**
- `POST /api/bookmarks` ‚Äî create a bookmark. Request body: `{"url":
  string, "title": string | null}`. Response 201: the created
  `domain.Bookmark` (JSON keys per its locked struct tags ‚Äî `id`,
  `original_url`, `canonical_url`, `identity_hash`, `title`, `tags`,
  `status`, `position`, `finished_at`, `author`, `created_at`,
  `updated_at`). Response 409: `{"error": "duplicate", "existing":
  <domain.Bookmark>}` when `RepositoryError.Kind == ErrKindDuplicate`.
  Response 400: `{"error": "invalid_url", "message": string}` when
  `Kind == ErrKindInvalidURL`.
- `GET /api/board` ‚Äî returns the full `domain.Board` (locked `inbox`/
  `reading`/`done` JSON keys). No query params in this issue (tag filter
  is Phase D).

**Optional endpoints:** none in this issue.

**Response shapes:** as above ‚Äî this issue is itself the source of truth
for these two shapes; there is no upstream third-party API to reference.

**Pagination semantics:** N/A ‚Äî `Board` returns the full board, no
pagination per PRD Non-Goals ("no pagination/infinite scroll").

**Developer confirmation required:** Developer must confirm the endpoint
layout above matches the intended Phase F implementation plan before the
Pre-Phase F test file is committed. If endpoints differ, this Wire
Contract is updated and the test fixture regenerated before the Dispatch
session opens.

## Acceptance Criteria

Given a valid, non-duplicate URL in a `POST /api/bookmarks` body
When the request is made
Then a 201 response returns the created bookmark's full JSON
representation, and the bookmark is retrievable via `GET /api/board`
immediately after

Given a duplicate URL in a `POST /api/bookmarks` body
When the request is made
Then a 409 response body contains the pre-existing bookmark's data

Given an invalid URL in a `POST /api/bookmarks` body
When the request is made
Then a 400 response is returned and no bookmark is created
